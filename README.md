# 6weeks-of-code

6 week programming challenge.

## Overview

This repository documents my 6-week coding journey.

The goal is simple:

* code every day for 6 weeks
* stay consistent
* work on one project
* improve logic, problem-solving, and programming fundamentals

This is not about speed.
This is about discipline, consistency, and getting better by building every day.

---

## Rules

1. Learn what is needed.
2. Code every day for 6 weeks.
3. Work on a single project.

---

## Challenge Rules

### 1. The Nopy Rule

Do not copy-paste code.
If code is needed, write it yourself.

### 2. The Noogle Rule

If stuck, do not Google immediately.
Try everything you can think of first.
If still stuck after 3 days, then Google it.

### 3. The Nooral Rule

Do not use neural networks (AI) such as ChatGPT, Gemini, etc.
Overusing them weakens problem-solving and independent thinking.

---

## Purpose

The purpose of this challenge is to:

* build consistency
* strengthen fundamentals
* improve coding logic
* develop problem-solving habits
* finish a real project through daily effort

## Focus

One project.
